## 知识点列表

### 1. 图的基本概念

| 属性 | 值 |
|------|-----|
| `name` | 图的基本概念 |
| `description` | 图 G=(V,E) 的定义、分类和基本术语 |
| `difficulty` | 2 |
| `category` | graph |
| `prerequisites` | [] |
| `subtopics` | ["有向图", "无向图", "完全图", "稀疏图", "稠密图"] |

**详细说明**：
- **顶点(Vertex)**：图中的节点，用 V 表示顶点集合
- **边(Edge)**：顶点之间的连接，用 E 表示边集合
- **有向图**：边有方向，记为 <u, v>
- **无向图**：边无方向，记为 (u, v)
- **完全图**：任意两个顶点之间都有边相连
- **度(Degree)**：顶点关联的边数（有向图分入度和出度）

### 2. 图的存储结构

| 属性 | 值 |
|------|-----|
| `name` | 图的存储结构 |
| `description` | 邻接矩阵、邻接表、十字链表等存储方式 |
| `difficulty` | 3 |
| `category` | graph |
| `prerequisites` | ["图的基本概念"] |
| `subtopics` | ["邻接矩阵", "邻接表", "邻接多重表", "十字链表"] |

**详细说明**：
- **邻接矩阵**：用 n×n 矩阵表示，matrix[i][j] 表示顶点 i 到 j 是否有边
  - 空间复杂度：O(n²)
  - 适用场景：稠密图
- **邻接表**：用链表数组表示，每个顶点对应一个链表存储相邻顶点
  - 空间复杂度：O(n+e)
  - 适用场景：稀疏图

### 3. 图的遍历

| 属性 | 值 |
|------|-----|
| `name` | 图的遍历 |
| `description` | 深度优先搜索(DFS)和广度优先搜索(BFS) |
| `difficulty` | 3 |
| `category` | graph |
| `prerequisites` | ["图的基本概念", "图的存储结构"] |
| `subtopics` | ["深度优先搜索", "广度优先搜索", "连通分量"] |

**详细说明**：
- **深度优先搜索(DFS)**：沿着路径尽可能深地探索，直到无法继续再回溯
  - 实现方式：递归或栈
  - 应用：拓扑排序、连通性检测、环检测
- **广度优先搜索(BFS)**：按层次逐层遍历，先访问距离起始顶点近的顶点
  - 实现方式：队列
  - 应用：最短路径（无权图）、二分图检测

## 复杂度分析

### 时间复杂度

| 操作 | 邻接矩阵 | 邻接表 |
|------|---------|--------|
| 判断两顶点是否相邻 | O(1) | O(degree(v)) |
| 遍历某顶点的所有邻接点 | O(n) | O(degree(v)) |
| 遍历整个图 | O(n²) | O(n+e) |
| DFS | O(n²) | O(n+e) |
| BFS | O(n²) | O(n+e) |

### 空间复杂度

| 存储方式 | 空间复杂度 |
|---------|-----------|
| 邻接矩阵 | O(n²) |
| 邻接表 | O(n+e) |

## 典型应用场景

1. **社交网络**：用户为顶点，关系为边，寻找好友推荐路径（BFS）
2. **地图导航**：城市为顶点，道路为边，寻找最短路径
3. **编译器**：函数调用图，检测循环依赖（DFS）
4. **网络爬虫**：网页为顶点，超链接为边，遍历网页（BFS/DFS）
